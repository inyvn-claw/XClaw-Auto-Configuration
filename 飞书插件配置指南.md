# 飞书插件配置指南

> 🦞 OpenClaw 飞书插件（openclaw-lark）完整配置说明

---

## 📋 配置目标

| 功能 | 默认行为 | 推荐配置 |
|------|---------|---------|
| 群聊触发 | 需要 @机器人 | 直接说话即可 |
| 私聊限制 | 仅白名单用户 | 所有人可用 |
| 连接稳定性 | 可能断开重连 | 持续稳定连接 |

---

## 一、前置条件

1. 已安装飞书插件：
   ```bash
   npx -y @larksuite/openclaw-lark install
   ```

2. 在[飞书开放平台](https://open.feishu.cn/)创建企业自建应用，获取：
   - **App ID**：`cli_xxxxxxxx`
   - **App Secret**：`xxxxxxxxxxxxxxxxxxxx`

3. 应用需开通以下权限：
   - `im:message` — 收发消息
   - `im:message.group_at_msg` — 群聊 @消息
   - `im:chat` — 群聊管理
   - `contact:user.base:readonly` — 用户信息

---

## 二、基础配置（~/.openclaw/openclaw.json）

### 1. 启用飞书插件

```json
{
  "plugins": {
    "entries": {
      "feishu": {
        "enabled": false
      },
      "openclaw-lark": {
        "enabled": true
      }
    },
    "allow": ["openclaw-lark"]
  }
}
```

### 2. 配置飞书渠道

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_你的AppID",
      "appSecret": "你的AppSecret",
      "domain": "feishu",
      "connectionMode": "websocket",
      "requireMention": false,
      "dmPolicy": "open",
      "allowFrom": ["*"],
      "groupPolicy": "open"
    }
  }
}
```

> ⚠️ **重要**：`appSecret` 必须直接写在配置文件中，不要使用外部文件引用（如 `{"source": "file"}`），否则每次消息触发时会被误判为"凭证变更"，导致 WebSocket 断开重连。

---

## 三、配置项说明

| 字段 | 说明 | 推荐值 |
|------|------|--------|
| `enabled` | 是否启用飞书渠道 | `true` |
| `appId` | 飞书应用 App ID | 从开放平台获取 |
| `appSecret` | 飞书应用密钥 | **直接内嵌，不要引用外部文件** |
| `connectionMode` | 连接方式 | `websocket`（推荐） |
| `requireMention` | 群聊是否必须 @机器人 | `false` |
| `dmPolicy` | 私聊策略 | `open`（所有人可用） |
| `allowFrom` | 允许的用户列表 | `["*"]`（配合 open 策略） |
| `groupPolicy` | 群聊策略 | `open` |
| `streaming` | 流式响应 | `true` |

## 四、解决"发一条就断"问题

### 问题日志

```
credentials changed, disposing stale instance
disconnecting WebSocket
ws client closed manually (force)
```

### 根本原因

使用外部文件存储凭证时，系统每次收到消息都重新读取文件，检测到"变化"后强制断开 WebSocket 重连。

### 解决方案

❌ **错误方式**（导致断开）：
```json
"appSecret": {
  "source": "file",
  "provider": "lark-secrets",
  "id": "/lark/appSecret"
}
```

✅ **正确方式**（稳定连接）：
```json
"appSecret": "直接写入密钥字符串"
```

## 🔑 常见问题速查

| 问题 | 解决方案 |
|------|---------|
| 需要 @才能触发 | `requireMention: false` |
| 私聊仅限白名单 | `dmPolicy: "open"` + `allowFrom: ["*"]` |
| 发一条就断开 | 直接内嵌 `appSecret`，不要引用外部文件 |
| 群聊受限 | `groupPolicy: "open"` |
| WebSocket 连接失败 | 检查 App ID/Secret 是否正确，重启 Gateway |
