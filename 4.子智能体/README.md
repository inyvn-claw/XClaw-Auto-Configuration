# 4. 子智能体 (Sub-agents)

本章节介绍 XClaw 的子智能体系统，用于科研 workflow 的自动化协作。

---

## 子智能体概述

子智能体（Sub-agents）是 XClaw 的核心功能之一，允许创建专门的 AI 代理来处理特定任务。通过多个子智能体的协作，可以构建完整的科研 workflow。

**核心优势：**
- 🎯 **专业化**：每个子智能体专注于特定领域
- 🤝 **协作性**：多个子智能体可以协同工作
- 🔄 **可复用**：子智能体可以在不同项目中重复使用
- 📊 **可管理**：统一的创建、监控和终止机制

---

## 各子智能体详细说明

### 1. 文献调研员 (researcher)

**角色定位**：科研信息的收集者和整理者

**核心职责**：
- 🔍 在学术数据库（Google Scholar, arXiv, PubMed 等）检索相关文献
- 📄 阅读并提取论文的核心观点、方法、实验结果
- 📝 生成文献综述和对比分析
- 📚 构建知识图谱，识别研究空白

**适用场景**：
- 新课题启动时的背景调研
- 撰写文献综述章节
- 跟踪领域最新进展

**输入**：研究主题、关键词、时间范围
**输出**：文献列表、摘要汇总、知识图谱

---

### 2. 创意生成器 (idea)

**角色定位**：创新思维的激发者

**核心职责**：
- 💡 基于现有研究提出创新点和改进方向
- 🔗 跨领域知识融合，发现新的研究机会
- 🎯 评估想法的可行性和潜在影响
- 📊 生成研究假设和初步方案

**适用场景**：
- 选题阶段的头脑风暴
- 论文修改时的创新点强化
- 寻找新的研究方向

**输入**：文献综述、现有问题、技术趋势
**输出**：创新点列表、研究假设、初步方案

---

### 3. 导师审核员 (mentor)

**角色定位**：学术质量的把关者

**核心职责**：
- ✅ 评估研究方案的科学性和合理性
- ⚠️ 识别潜在风险和局限性
- 📈 提供改进建议和优化方向
- 🎓 从资深研究者角度提供指导

**适用场景**：
- 开题报告审核
- 实验方案评审
- 论文投稿前的质量把关

**输入**：研究方案、实验设计、初步结果
**输出**：评审意见、改进建议、风险预警

---

### 4. 架构设计师 (architect)

**角色定位**：技术方案的规划者

**核心职责**：
- 🏗️ 设计系统整体架构和模块划分
- 🛠️ 技术选型和工具评估
- 📐 定义接口规范和数据流
- 🔧 制定实现路线图

**适用场景**：
- 复杂系统的技术方案设计
- 算法框架的模块化设计
- 实验平台的架构规划

**输入**：功能需求、性能指标、约束条件
**输出**：架构图、技术选型文档、实现计划

---

### 5. 实验工程师 (coder)

**角色定位**：代码和实验的实现者

**核心职责**：
- 💻 根据设计方案编写高质量代码
- 🧪 搭建实验环境，执行实验
- 🐛 调试代码，修复错误
- 📊 收集和分析实验数据

**适用场景**：
- 算法实现和优化
- 实验代码开发
- 数据处理和可视化

**输入**：架构设计、算法描述、数据集
**输出**：可运行代码、实验结果、技术文档

---

### 6. 论文撰写员 (writer)

**角色定位**：学术内容的创作者

**核心职责**：
- ✍️ 撰写学术论文各章节（摘要、引言、方法、结果、讨论等）
- 🎨 遵循学术写作规范和格式要求
- 🌐 语言润色，确保表达清晰准确
- 📋 生成图表、表格和参考文献

**适用场景**：
- 论文初稿撰写
- 章节修改和润色
- 多语言翻译和本地化

**输入**：实验结果、方法描述、相关文献
**输出**：论文草稿、图表、参考文献列表

---

### 7. 论文审稿人 (reviewer)

**角色定位**：论文质量的审查者

**核心职责**：
- 🔍 检查论文的逻辑结构和论证过程
- 📏 评估方法的合理性和实验的充分性
- 📝 审查语言表达和格式规范
- 💬 提供详细的修改建议和评审意见

**适用场景**：
- 投稿前的自我审查
- 模拟同行评审
- 多轮修改的质量控制

**输入**：论文草稿、审稿标准
**输出**：审稿意见、修改建议清单、质量评分

---

### 8. 科研主管 (coordinator)

**角色定位**：科研项目的协调者

**核心职责**：
- 📋 制定项目计划和里程碑
- 👥 分配任务给其他子智能体
- 📊 跟踪项目进度，识别瓶颈
- 🔄 协调各子智能体之间的协作

**适用场景**：
- 大型科研项目的整体管理
- 多阶段 workflow 的编排
- 资源分配和优先级管理

**输入**：项目目标、资源约束、时间要求
**输出**：项目计划、任务分配、进度报告

## 使用示例

### 启动子智能体

```bash
# 启动文献调研员
sessions_spawn({
  "task": "调研大语言模型在代码生成领域的最新进展",
  "agentId": "researcher",
  "mode": "session",
  "label": "paper-research-001"
})

# 启动实验工程师
sessions_spawn({
  "task": "实现 Transformer 模型的训练代码",
  "agentId": "coder",
  "mode": "session",
  "label": "experiment-coding-001"
})
```

### 子智能体协作

```
用户：我想写一篇关于多模态学习的论文

Coordinator: 好的，我来协调这个任务
  ├── 分配 researcher：调研多模态学习最新进展
  ├── 等待 researcher 完成...
  ├── 分配 idea：基于调研结果提出创新点
  ├── 分配 mentor：审核研究方向的可行性
  ├── 分配 architect：设计实验架构
  ├── 分配 coder：实现实验代码
  ├── 分配 writer：撰写论文
  └── 分配 reviewer：审稿并提出修改意见
```

- 全局配置：`~/.openclaw/openclaw.json`

## 故障排查

### 子智能体无法启动

**问题**：`sessions_spawn` 返回错误

**解决方案**：
1. 检查 `agentId` 是否正确配置
2. 确认模型名称有效
3. 查看 OpenClaw 日志获取详细错误信息

### 子智能体任务超时

**问题**：子智能体长时间无响应

**解决方案**：
1. 使用 `subagents list` 查看状态
2. 使用 `sessions_history` 查看执行历史
3. 必要时使用 `subagents kill` 终止并重试

### 子智能体协作不畅

**问题**：多个子智能体之间信息传递不顺畅

**解决方案**：
1. 确保 coordinator 正确分配任务
2. 使用明确的输入输出格式
3. 在中间步骤添加检查点
